# Laravel Deployment Guide

## 1. Build frontend assets locally
Production assets must be compiled before uploading the project.

```bash
npm install
npm run build
```
This generates optimized files in:

```
public/build/
```
These are the only Vite assets used in production.

---

## 2. Upload the Laravel project to the server
On cPanel, place the Laravel project **outside** `public_html`:

```
/home/USER/
    laravel-app/
        app/
        bootstrap/
        config/
        public/
        resources/
        routes/
        vendor/
        ...
```
Only the contents of `public/` go into `public_html`.

---

## 3. Move public files into public_html
Upload everything from:

```
laravel-app/public/
```
into:

```
public_html/
```
This includes:
- `index.php`
- `build/` (generated by Vite)
- `assets/` (if any)
- `favicon`, `robots.txt`, etc.

---

## 4. Fix index.php paths
Edit:

```
public_html/index.php
```
Update the autoload and bootstrap paths:

```php
require __DIR__.'/../laravel-app/vendor/autoload.php';
$app = require_once __DIR__.'/../laravel-app/bootstrap/app.php';
```
Replace `laravel-app` with your actual folder name.

---

## 5. Upload the .env file
Place your `.env` file in:

```
/home/USER/laravel-app/.env
```
Update:

```
APP_ENV=production
APP_DEBUG=false
APP_URL=https://yourdomain.com
```

---

## 6. Run Laravel optimization commands
Open cPanel Terminal and run:

```bash
php artisan key:generate
php artisan optimize
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

---

## 7. Create the storage symlink
Still in Terminal:

```bash
php artisan storage:link
```
This makes `/storage` accessible from the web.

---

## 8. Set PHP version and extensions
In cPanel → Select PHP Version:

- PHP 8.1 or 8.2
- Enable:
  - mbstring
  - openssl
  - pdo_mysql
  - tokenizer
  - xml
  - fileinfo
  - curl

---

## 9. Configure the domain
In cPanel → Domains:

Document Root must be:

```
public_html/
```

---

## 10. Folder structure overview

```
/home/USER/
    laravel-app/
        app/
        bootstrap/
        config/
        resources/
        routes/
        vendor/
        .env
        artisan
        ...

public_html/
    index.php
    build/
    assets/
    favicon.ico
    robots.txt
```

---

## 11. Deployment workflow summary
- Build assets locally: `npm run build`
- Upload Laravel project (except `public/`) outside `public_html`
- Upload `public/` contents into `public_html`
- Fix `index.php` paths
- Upload `.env`
- Run artisan optimize commands
- Create storage symlink
- Ensure PHP 8.1+ and required extensions
- Set domain root to `public_html`
